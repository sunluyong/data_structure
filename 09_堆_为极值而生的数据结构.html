算法与数据结构

堆：为极值而生的数据结构

从 BST 到 堆 (Heap)

BST：全能选手

BST 擅长查找任意值，保持全局有序。

查找 Max/Min 复杂度：$O(\log N)$

堆：极值专家

如果业务只关心最大值/最小值

  • 查找 Max/Min 复杂度：$O(1)$。
  • 并不保证整体有序，只保证父子有序。

"我不需要知道第二名是谁，
我只要找到第一！"

堆的基础概念

两个核心性质

  • 1. 结构性：完全二叉树

    除最后一层外全满，最后一层节点靠左

  • 2. 有序性：堆序性

    任意节点的值 $\ge$ (或 $\le$) 其子节点

    • 大顶堆 (Max Heap): 根最大 (Parent $\ge$ Child)
    • 小顶堆 (Min Heap): 根最小 (Parent $\le$ Child)

物理存储：数组实现

完全二叉树可以完美映射到数组，无需链表和指针。

索引关系 (0-based)

  • Parent(i) = floor((i - 1) / 2)
  • Left(i)   = 2*i + 1
  • Right(i)  = 2*i + 2

核心操作：节点上浮与下沉

上浮 (Sift Up/ Swim)

小根堆中某个节点“太小了”，比父亲还小，应该往上爬，直到不再比父亲小。

典型出现：
插入（新元素先放在数组末尾/树的最后一个位置）。

下沉 (Sift Down/ Sink)

小根堆某个节点“太大了”，比孩子还要大，应该往下沉，直到不必孩子大。

典型出现：
删除堆顶（把最后一个元素搬到根，根变大了，需要往下修）。

插入 (Push)

公式：Append + Swim

  1. Append：将新元素放到数组末尾。
  2. Swim：执行上浮操作恢复堆序。

时间复杂度：$O(\log N)$

2. 动画演示：插入 99

// 辅助宏定义 #define PARENT(i) ((i - 1) / 2) // 插入操作：修改 size 需要传指针 void push(int* heap, int* size, int val) { // 1. 放到末尾 (先赋值再 +1) heap[(*size)++] = val; // 2. 上浮恢复堆序 swim(heap, *size - 1); } // 上浮：只是一个原子操作 void swim(int* heap, int cur) { while (cur > 0 && heap[cur] > heap[PARENT(cur)]) { swap(&heap[cur], &heap[PARENT(cur)]); cur = PARENT(cur); } }

删除堆顶 (Pop)

公式：Replace + Sink

  1. Remove：移除堆顶（拿到最大值）。
  2. Replace：用末尾元素补到堆顶。
  3. Sink：执行下沉操作恢复堆序。

时间复杂度：$O(\log N)$

2. 动画演示：Pop Max

// 辅助宏定义 #define LEFT(i) (2 * i + 1) int pop(int* heap, int* size) { int ret = heap[0]; // 1. 末尾补到堆顶 heap[0] = heap[--(*size)]; // 2. 下沉恢复堆序 sink(heap, *size, 0); return ret; } void sink(int* heap, int n, int i) { while (LEFT(i) < n) { int child = LEFT(i); // 选较大的孩子 if (child + 1 < n && heap[child+1] > heap[child]) child++; if (heap[i] >= heap[child]) break; swap(&heap[i], &heap[child]); i = child; } }

建堆：插入与下沉的效率

方案 A：逐个插入 (Swim)

从空堆开始，依次对 N 个元素执行 Push 操作

  • 新元素加在末尾，需要一路“上浮”到根
  • 大部分节点（叶子）都要经历漫长的上浮路径

复杂度：$O(N \log N)$

方案 B：批量下沉 (Sink)

原地调整：从最后一个非叶子节点开始，倒序遍历执行 Sink

  • 利用特性：叶子节点（占 $N/2$）天然有序，无需操作
  • 越往上层节点越少，需要操作的节点数呈指数级减少，数学级数求和收敛于 $O(N)$

复杂度：$O(N)$ (更优)

演示：下沉法建堆 ($O(N)$)

Ready

建堆代码实现

从最后一个非叶子节点开始，倒序执行下沉

  • 起点：i = n / 2 - 1
  • 方向：i 递减到 0
  • 操作：对每个 i 执行 sink(heap, n, i)

为什么是 $O(N)$？
因为叶子节点（占一半）不需要下沉；倒数第二层最多下沉 1 层...
节点越少，下沉深度越深；节点越多，下沉深度越浅。
数学证明：$\sum \frac{n}{2^{h+1}} \cdot h \approx O(n)$

// 下沉操作 (复用之前的 sink 函数) void sink(int* heap, int n, int i) { while (2*i + 1 < n) { int child = 2*i + 1; // 选较大的孩子 if (child + 1 < n && heap[child+1] > heap[child]) child++; if (heap[i] >= heap[child]) break; swap(&heap[i], &heap[child]); i = child; } } // 建堆：O(N) void build_heap(int* heap, int n) { // 从最后一个非叶子节点开始 for (int i = n / 2 - 1; i >= 0; i--) { sink(heap, n, i); } }

递归实现建堆

递归版本的建堆，代码更简洁优雅

  • 递归 sink：比较当前节点与子节点，交换后递归处理
  • 迭代建堆：从最后一个非叶子节点开始，顺序调整
  • 复杂度：仍然是 $O(N)$，递归深度 $O(\log N)$

递归 vs 迭代
递归版本代码更易读，但可能存在栈溢出风险（深度 $\log N$）。
迭代版本更安全，是生产环境的首选。

// 递归下沉操作 void heapify(int* heap, int n, int i) { int largest = i; int left = 2 * i + 1; int right = 2 * i + 2; // 找出最大节点 if (left < n && heap[left] > heap[largest]) largest = left; if (right < n && heap[right] > heap[largest]) largest = right; // 如果最大不是当前节点，交换并递归 if (largest != i) { swap(&heap[i], &heap[largest]); heapify(heap, n, largest); } } // 递归建堆：O(N) void build_heap(int* heap, int n) { // 从最后一个非叶子节点开始 for (int i = n / 2 - 1; i >= 0; i--) { heapify(heap, n, i); } }
递归让逻辑更清晰

堆排序 (Heap Sort)

  1. 建堆 (Build)：
    先将数组转化为大顶堆 ($O(N)$)
  2. 交换 (Swap)：
    将堆顶与末尾互换 -> 最大值归位
  3. 下沉 (Sink)：
    堆大小减 1，剩余部分执行下沉恢复堆序
  4. 循环：
    重复步骤 2-3，直到堆为空

时间： $O(N \log N)$

空间： $O(1)$ (原地)

void heap_sort(int* arr, int n) { // 1. 建堆 (变成大顶堆) for (int i = n/2 - 1; i >= 0; i--) sink(arr, n, i); // 2. 排序 (Swap + Sink) for (int i = n - 1; i > 0; i--) { // 把最大值(堆顶)换到末尾 swap(&arr[0], &arr[i]); // 剩余元素(大小为 i) 恢复堆序 sink(arr, i, 0); } }

堆的适用场景

优先队列

Priority Queue

普通队列是 FIFO，优先队列是 BIFO (Best-In First-Out)。

  • 谁优先级高，谁先出队。
  • 插入 $O(\log N)$，取出 $O(\log N)$。

例子：操作系统任务调度

Top-K 问题

海量数据处理

从 N 个元素中找到前 K 大/小的元素。

  • 排序法：$O(N \log N)$
  • 堆解法：$O(N \log K)$ (维护大小为 K 的堆)

例子：微博热搜、游戏榜单

图算法加速

Dijkstra & Prim

在图论算法中，利用堆快速获取“当前距离最近”的节点。

  • Dijkstra：寻找最短路径。
  • 性能提升：将查找最小值从 $O(V)$ 优化到 $O(1)$。

例子：地图导航、网络路由

LeetCode 295: 数据流的中位数

问题：动态添加数字，随时返回中位数。

双堆法思路

维护两个堆，各存一半数据：

  • 大顶堆 (MaxHeap)：存较小的一半数。
    堆顶是这一半的最大值（左边最靠近中间的）。
  • 小顶堆 (MinHeap)：存较大的一半数。
    堆顶是这一半的最小值（右边最靠近中间的）。

中位数 = 两个堆顶的平均值 (或元素多的那个堆顶)。

代码实现：双堆求中位数

宏观设计

把数据流切成两半，保持有序：

[ 小数区 ... Max ] <-- 中位数 --> [ Min ... 大数区 ]
  • 左边 (MaxHeap)：存较小的一半。
  • 右边 (MinHeap)：存较大的一半。
  • 规定：左边数量 $\ge$ 右边 (最多多1个)。

如果不比较直接插入，可能会把“大数”错放到左边。
策略：新来的先去左边“过一下”，选出最大的送去右边；如果右边人多了，再匀一个回左边。

// 假设已封装堆操作: push_max, pop_max, push_min, pop_min void addNum(MedianFinder* obj, int num) { /* 1. 筛选：新元素必须经过"比较"才能去右边 */ // 先扔进左边(大顶堆) push_max(obj->left, num); // 选出左边最大的，送给右边(小顶堆) int max_val = pop_max(obj->left); push_min(obj->right, max_val); /* 2. 平衡：右边不能比左边多 */ if (obj->right_size > obj->left_size) { // 把右边最小的拿出来，还给左边 int min_val = pop_min(obj->right); push_max(obj->left, min_val); } } double findMedian(MedianFinder* obj) { // 如果左边多 1 个，中位数就是左边堆顶 if (obj->left_size > obj->right_size) { return peek_max(obj->left); } // 否则是两个堆顶的平均值 return (peek_max(obj->left) + peek_min(obj->right)) / 2.0; }

巅峰对决：堆 VS BST

维度 堆 (Heap) 二叉搜索树 (BST)
主要目标 快速找极值 (Top 1) 快速找任意值、排序
查找 Max/Min $O(1)$ $O(\log N)$ (或者 $O(H)$)
查找任意值 $O(N)$ (无序，只能遍历) $O(\log N)$
有序性 弱序 (仅父子有序) 强序 (左 < 根 < 右)
应用场景 优先队列、Top K、中位数 Set/Map 实现、数据库索引

课堂小结

核心回顾

  • 数据结构：完全二叉树 + 堆序性。
  • 存储：数组 (Index 0-based)。
  • 关键算法：
    • Swim (插入): $O(\log N)$
    • Sink (删除/建堆): $O(\log N)$
    • Heapify: $O(N)$
  • 适用性：只看极值，不看整体。
