什么是算法 (Algorithm)？

算法是完成一项任务或解决一个问题的明确、有限的步骤和流程。

算法无处不在：

  • 旅行计划
    制定路线的步骤
  • 烹饪食谱
    做菜的流程
  • 查字典
    找字的策略

直观对比：如何查字典？

❌ 方式 A：无算法

从第一页开始，一页一页往后翻，直到找到那个字。

// 效率极低
// 假如字典有 2000 页，可能要翻 2000 次

✅ 方式 B：高效算法
  • 快速定位：根据拼音索引快速定位大致范围
  • 折半查找：每次翻一半，决定是往前还是往后

// 效率极高
// 2000 页最多只需要翻 11 次

什么是数据结构 (Data Structure)？

思考一个问题：

如果报纸散落一地（乱序），你还能用“折半查找”快速找到某条新闻吗？

不能！

只能从头看到尾（退化为暴力查找）。

数据结构是算法高效运行的前提。

它是特意设计的数据存储和组织方式。

  • 数据结构 = 字典排版（拼音排序、索引）
  • 算法 = 查字典方法（二分查找）

只有将数据精心组织（构建结构），高效的算法才有施展的舞台。

Program = Data Structure + Algorithm

—— Niklaus Wirth (图灵奖得主)

程序 = 特定的数据结构 + 解决问题的步骤

数据结构与算法无处不在

每一个顺滑的 App 功能背后，都有特定的数据结构在支撑。

朋友圈 / 社交网络

为什么能算出“共同好友”？

核心结构：图 (Graph)

人是节点，关系是连线。

浏览器“后退”

为什么总是回到上一个页面？

核心结构：栈 (Stack)

先进后出 (LIFO) 的容器。

数据结构与算法将学什么？

计算机对数据的核心操作：

  • 增 (Insert)
  • 删 (Delete)
  • 改 (Update)
  • 查 (Search)
课程三大模块
  1. 构建 (数据结构)
    线性表、栈、队列、树、图
  2. 治理 (排序算法)
    让无序变有序，为高效铺路
  3. 检索 (查找算法)
    在特定结构中，精准定位目标
